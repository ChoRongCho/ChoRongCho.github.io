2-1. Problem Formulation

마지막 업데이트: 2025-10-03

Transition System

Definition

Classical AI Planning 문제에서 transition system 혹은 state space는 다음과 같이 정의됩니다:

\[ \Theta = (S, L, T, c, I, S^G) \]

  • \(S\): a finite set of states
  • \(L\): a finite set of trasnsition label
  • \(T \subseteq S \times L \times S \) : a transition relation
  • \(c: L \to \mathbb{R}^+_0\): the cost function
  • \(I \in S \): the initial state
  • \(S^G \subseteq S \): the set of goal states

Terminology

State의 갯수를 \(\Theta\)의 크기(size)라 부릅니다. 그리고 모든 state \(s\)와 label \(l\)에 대해서 다음 상태 \(s^\prime\)이 최대 1개만 존재하면, \(\Theta\)는 결정론적(deterministic)이라고 말합니다. 그리고 모든 label에 대해서 \(c(l)=1\)이면 unit cost라고 부릅니다.

어떤 상태 \(s^\prime\)이 상태 \(s\)로부터 일련의 전이(sequence of transitions)를 통해 도달 가능하면, \(s^\prime\)은 \(s\)로부터 reachable하다고 합니다. 이때 전이의 나열 \(l_1, \dots, l_n\)을 \(s\)에서 \(s^\prime\)까지의 path라 부르고, 그 경로의 총 비용은 \(\sum_{i=1}^n c(l_i)\)로 정의합니다. 초기 상태 \(I\)에서 목표 상태 집합 \(S_G\)의 어떤 상태에 도달하는 path를 solution이라 하며, 그 중 비용이 최소인 것이 optimal solution입니다. 만약 어떤 상태 \(s\)에서 solution이 존재하지 않는다면, \(s\)는 dead end라 부릅니다. 초기 상태 \(I\)에서 solution이 존재하면 \(\Theta\)는 solvable하다고 합니다.

STRIPS

STRIPS (Stanford Research Institute Problem Solver)는 1971년 Fikes와 Nilsson이 제안한, planning problem \(\Pi\)를 표현하기 위한 가장 대표적인 계획 형식(planning formalism)입니다.

STRIPS는 \(\Pi\)를 문제를 다음과 같이 표현합니다.

  • \(P\): a finite set of facts, also propositions
  • \(A\): a finite set of action
  • \(c: L \to \mathbb{R}^+_0\): the cost function
  • \(I \subseteq P \): the initial state
  • \(S^G \subseteq S \): the goal

\[ \Pi = (P, A, c, I, G) \]

STRIPS에서 하나의 액션은 다음과 같이 구성됩니다:

  • Preconditions: 액션을 실행하기 위해 반드시 만족해야 하는 조건들의 집합
  • Add list: 액션 실행 후 새롭게 참이 되는 사실들
  • Delete list: 액션 실행 후 더 이상 참이 아니게 되는 사실들

STRIPS의 핵심 아이디어는 액션이 상태를 변환하는 규칙으로 정의된다는 점입니다. 즉, 액션을 적용하면 현재 상태에서 Preconditions가 충족될 때만 실행 가능하며, 실행 결과로 Add list는 상태에 추가되고, Delete list는 상태에서 제거됩니다.

  • Initial State – 현재 세계의 상태를 나타냄
  • Actions – 적용 가능한 연산자(전제조건 + 효과)
  • Goal State – 달성해야 할 조건 집합

예시

예를 들어, Blocks World에서
초기 상태는 블록 A, B, C가 테이블 위에 흩어져 있는 상황,
목표 상태는 A를 B 위에 두고, B를 C 위에 두는 상태일 수 있습니다.
